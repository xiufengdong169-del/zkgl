# 第一阶段架构基线

## 新建系统原则

本项目全部为新开发程序。前端、云函数 API、数据库结构、权限体系和部署配置均从零建设，不接续旧系统，不复用旧程序，不兼容旧接口，也不迁移历史数据库或存量数据。

## 安全边界

浏览器仅使用 CloudBase 环境 ID 和 Publishable Key，完成用户名密码登录并取得用户访问令牌。浏览器不得接触 MySQL 密码、SecretKey 或服务端 API Key。

云函数 API 负责再次校验 CloudBase 身份、内部 `iam_user` 启用状态、功能权限和数据范围。用户、业务记录、审批任务、状态历史与审计日志的同一业务动作必须在同一数据库事务内提交。

## 权限判定顺序

1. CloudBase 会话有效。
2. `cloudbase_uid` 能唯一映射到启用的内部用户。
3. 用户具有操作所需的功能权限码。
4. 数据对象满足 ALL、部门、项目或本人范围之一。
5. 敏感字段和附件再执行独立授权检查。

任一步失败均拒绝请求，并写入不包含密码、令牌或密钥的审计记录。

## 数据库初始化

本项目没有历史数据库和存量数据迁移。开发、测试及首次上线环境均从空库执行 `database/init/schema.sql` 完成一次性初始化，不建设迁移版本表或迁移执行器。正式投产后的结构变更机制须单独评审，不预先纳入本阶段范围。
